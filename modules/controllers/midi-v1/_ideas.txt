create these receiver classes dereived from receiver_base 
class receiver_noteOnOff:
playing intents like an instrument. note-off is translated to note-on with velocity 0. when note-on is received it's velocity has to be within a given range to be considered. Then velocityOffset is added and multiplied by velocityScale. Then a final function curve is applied for each target and then a runtime update is sent.

class receiver_noteAndControl:
note-on selects a state (in memory) for this note/channel and waits for a given controller number to arrive, which then sends the value to targets after function is applied until note-off is received.
